# Code Style and Conventions

This document defines the canonical code style and conventions for the resume monorepo.

## Overview

JavaScript primary with TypeScript for type annotations only. ESM in all apps and packages, CJS for root config files. Node >= 22 required.

## Language and Runtime

- JavaScript primary (.js files), TypeScript for type annotations only (.ts files)
- Node >= 22 required (enforced by `.nvmrc`)
- ESM (`"type": "module"`) in all apps and packages
- CJS only for root config files (eslint.config.cjs, jest.config.cjs, playwright.config.js)
- Target: ES2022 (from jsconfig.json)

## File and Directory Naming

- Directories: kebab-case (e.g., `apps/portfolio/lib/`)
- Files: lowercase with dots/hyphens allowed (e.g., `generate-worker.js`)
- Contract files (uppercase): AGENTS.md, README.md, CHANGELOG.md, LICENSE
- Path aliases (from jsconfig.json):
  - `@lib/*` → `apps/portfolio/lib/*`
  - `@tests/*` → `tests/*`

## ESLint Rules

Key rules from eslint.config.cjs (ESLint 9 Flat Config):

| Rule            | Setting                          |
| --------------- | -------------------------------- |
| semicolons      | required                         |
| quotes          | single quotes, avoidEscape: true |
| no-var          | error                            |
| prefer-const    | warn                             |
| eqeqeq          | always, null: ignore             |
| no-eval         | error                            |
| no-implied-eval | error                            |
| no-new-func     | error (test files excepted)      |
| no-unused-vars  | warn (argsIgnorePattern: `^_`)   |

Test files (`tests/**/*.test.js`, `tests/**/*.spec.js`) have no-new-func disabled.

Ignored patterns: node_modules, build outputs (worker.js, dist), coverage, logs, .wrangler.

## Module Patterns

### Portfolio (apps/portfolio/)

- Pure functions in `lib/` — receive env, produce output, no side effects
- Build-time asset inlining — zero runtime I/O
- CSS variables for theming (see `src/styles/variables.css`)
- `worker.js` is generated by `generate-worker.js` — never edit directly
- Multi-language: `data.json` (ko), `data_en.json`, `data_ja.json`

### Job Server (apps/job-server/)

- Hexagonal architecture: services (domain logic) ↔ clients (external adapters)
- DI via constructor injection, no global state
- MCP tool export pattern:
  ```js
  export const {name}Tool = { ... }
  ```
- Fastify route pattern:
  ```js
  export default async function nameRoutes(fastify) { ... }
  ```
- Import from `src/shared/` directly — never import from `lib/` (deprecated)

### Job Dashboard (apps/job-dashboard/)

- All handlers extend `BaseHandler(db, cache, env)`
- KV entries MUST have TTL
- Rate limiting: 60 req/min per IP per endpoint

## Testing Conventions

| Test Type   | Location                           | Framework  | Notes                |
| ----------- | ---------------------------------- | ---------- | -------------------- |
| Unit        | `tests/unit/`                      | Jest       | 33 suites, 712 tests |
| Integration | `apps/job-server/src/**/*.test.js` | node:test  | 93 tests             |
| E2E         | `tests/e2e/`                       | Playwright | 24 files             |

Coverage (via c8): lines 60%, functions 55%, branches 70%.

Critical: Never use `networkidle` in Playwright — use `domcontentloaded`.

## Commit and PR Standards

- Conventional Commits: `type(scope): imperative summary` (<=72 chars, lowercase)
- Types: `feat`, `fix`, `docs`, `refactor`, `test`, `ci`, `chore`, `perf`, `build`, `revert`
- Squash merge only
- ~200 LOC max per PR
- SHA-pin all GitHub Actions with `# vN` version comment (e.g., `actions/checkout@v4 # v4.2.2`)

## Anti-Patterns

| Pattern                                           | Reason                        |
| ------------------------------------------------- | ----------------------------- |
| `as any`, `@ts-ignore`, `@ts-expect-error`        | Type safety                   |
| Empty catch blocks (`catch {}`)                   | Error swallowing              |
| Hardcoded credentials, resume IDs, session tokens | Security                      |
| KV entries without TTL                            | Resource leak                 |
| Mutable action tags (`@v4`)                       | Reproducibility               |
| Naked Puppeteer without stealth                   | Detection risk                |
| Direct edits to `worker.js`                       | Build artifact (regenerated)  |
| Merge commits                                     | Squash only                   |
| `lib/` imports in job-server                      | Deprecated, use `src/shared/` |
| Manual `npm run deploy`                           | Blocked, use git push         |

## CI Pipeline

Eight-job validation pipeline:

1. analyze — dependency review
2. validate-cf — Wrangler config validation
3. lint — ESLint
4. typecheck — tsc --noEmit
5. test-unit — Jest
6. test-e2e — Playwright
7. security-scan — npm audit
8. build → elk-ingest — worker.js build + ELK logging

Deploy authority: Cloudflare Workers Builds (git push to master). GitHub Actions is CI only.
