# ADR Template

- Status: Proposed
- Date: YYYY-MM-DD
- Deciders: Core maintainers
- Technical Story: Link to issue or PR

## Context

Describe the problem, constraints, and forces that require a decision.

## Decision

Describe the chosen approach and the key architectural trade-offs.

## Consequences

- Positive: Expected benefits and rationale.
- Negative: Known drawbacks, risks, and costs.
- Follow-up: Required implementation or migration actions.
